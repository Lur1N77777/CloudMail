# 2026-04-19 Admin UI Redesign Design

## 背景
当前 `app/admin.tsx` 功能已初步接通，但视觉上更接近“后台控制台”，与现有移动端邮件 UI（`app/(tabs)/index.tsx` + `app/mail-detail.tsx`）风格割裂。用户希望管理员模式在保持超级权限的同时，拥有与普通邮件界面接近的统一体验，并支持邮件内容完整查看、主题与正文预览、验证码快捷复制。

## 备选方案

### 方案 A（采用）
管理员模式使用“原 UI 同构”的信息架构：顶部 header + 轻量分段 tabs + 邮件列表 + 详情页。
- 列表项复用普通收件箱的视觉语言：头像、发件人/收件人、日期、主题、正文预览。
- 管理员详情页延续 `mail-detail` 的阅读体验，增加管理员专属元信息和快捷操作。
- 对验证码邮件在列表和详情中都提供快捷复制。

优点：与现有 App 最一致，学习成本低，移动端最好用，风险最低。
缺点：需要新增管理员详情页与少量共享工具。

### 方案 B
管理员列表支持“行内展开全文”。
优点：少一次跳转。
缺点：列表会过长，复杂邮件阅读体验差，不像现有 UI。

### 方案 C
管理员列表预览 + 底部弹层预览全文。
优点：现代感较强。
缺点：与当前产品既有交互不一致，复杂正文/原文信息展示受限。

## 最终设计
采用方案 A。

### 1. 信息架构
- `app/admin.tsx`
  - 保留管理员模式入口与分段导航，但整体压缩为更接近现有“邮件”页的 header/segmented 风格。
  - `mails/sendbox/unknown` 三类邮件使用统一的列表式呈现。
- 新增 `app/admin-mail-detail.tsx`
  - 用于查看管理员选中的任意邮件。
  - 阅读结构参考 `app/mail-detail.tsx`，但补充管理员专属字段（邮箱归属、邮件来源类型、原始发件/收件信息、验证码卡片）。

### 2. 列表视觉与交互
- 邮件列表项包含：
  - 左侧圆形头像（发件人首字母/目标收件人首字母）
  - 第一行：主要身份 + 日期
  - 第二行：主题
  - 第三行：正文预览（从 text/html 提取纯文本）
  - 辅助信息：邮箱归属地址、未知收件标记、附件图标
- 若识别到验证码：
  - 列表项右下或底部显示一个轻量 code pill，例如 `验证码 123456`
  - 点击 pill 直接复制，不必进入详情
- 点击列表项：进入详情页
- 长按列表项：维持管理员删除操作

### 3. 详情页设计
- 顶部保持与 `mail-detail.tsx` 一致的阅读头部：返回、删除
- 正文前增加管理员信息卡：
  - 当前邮件类型（收件/发件/未知）
  - 所属地址（如果后端提供）
  - 发件人 / 收件人 / 时间
- 如果识别到验证码：
  - 在主题下方插入醒目的验证码卡片
  - 显示验证码、识别说明、复制按钮
- 正文区域：
  - 优先展示 `text`
  - 若只有 `html`，先做纯文本清洗后展示
  - 保持可选中复制
- 保留附件展示

### 4. 验证码识别规则
- 在 `lib/mail-parser.ts` 增加轻量工具：
  - 从 text/html 中提取纯文本正文
  - 根据关键词（验证码、verification code、OTP、code、login code 等）+ 数字/字母数字模式识别 4-8 位验证码
- 识别结果尽量保守：
  - 优先选择最像验证码的短 token
  - 不追求 100% 覆盖，但要避免把普通订单号误判得太夸张

### 5. 一致性原则
- 不把管理员页继续做成“后台 dashboard 卡片风”
- 颜色、圆角、间距、header、list row、详情阅读体验尽量靠近已有页
- 管理员功能强，但界面仍然是“邮件客户端的一部分”

### 6. 验证要求
- TypeScript 必须通过
- Admin 相关页面 ESLint 至少无新增 error
- Android release 构建成功
- 手动验证路径：管理员页 → 列表预览 → 进入详情 → 复制验证码
