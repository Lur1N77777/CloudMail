# CloudMail 设计文档

## 应用概述

CloudMail 是一个基于 Cloudflare 临时邮箱后端的移动客户端应用。用户可以通过填入自己的 Cloudflare Worker 地址和认证信息，连接到自建的临时邮箱服务，实现邮件的收发和管理。

## 屏幕列表

| 屏幕名称 | 描述 | Tab |
|---------|------|-----|
| 收件箱 (Inbox) | 邮件列表，支持下拉刷新和自动轮询 | Tab 1 |
| 发送邮件 (Compose) | 撰写和发送邮件 | Tab 2 |
| 邮箱管理 (Addresses) | 管理邮箱地址，创建新地址 | Tab 3 |
| 设置 (Settings) | 服务器配置、应用设置 | Tab 4 |
| 邮件详情 (Mail Detail) | 查看邮件正文、附件 | Stack Screen |

## 主要内容和功能

### 收件箱 (Inbox)
- 邮件列表卡片：发件人、主题、时间、摘要预览
- 下拉刷新 + 自动轮询（可配置间隔 10/30/60 秒）
- 左滑删除邮件
- 点击进入邮件详情
- 顶部显示当前邮箱地址，可快速切换
- 空状态提示

### 邮件详情 (Mail Detail)
- 邮件头信息：发件人、收件人、主题、时间
- HTML 邮件正文渲染（WebView）
- 纯文本邮件正文
- 附件列表和下载

### 发送邮件 (Compose)
- 发件人名称输入
- 收件人地址和名称输入
- 主题输入
- 正文编辑（支持纯文本/HTML 切换）
- 发送按钮

### 邮箱管理 (Addresses)
- 当前活跃邮箱地址显示
- 创建新邮箱地址（名称 + 域名选择）
- 已保存的邮箱地址列表
- 切换活跃邮箱
- 删除邮箱地址
- JWT 凭证显示/复制

### 设置 (Settings)
- 服务器地址 (Worker URL)
- Admin 密码（可选，用于 Admin API）
- 站点访问密码（可选）
- 自动刷新间隔设置
- 深色/浅色模式切换
- 关于信息

## 关键用户流程

### 首次使用
1. 用户打开应用 → 设置页面
2. 填入 Worker URL → 保存
3. 可选填入 Admin 密码和站点密码
4. 切换到邮箱管理 → 创建新邮箱
5. 输入邮箱名称，选择域名 → 创建
6. 自动切换到收件箱，开始接收邮件

### 查看邮件
1. 收件箱列表 → 点击邮件卡片
2. 进入邮件详情页
3. 查看邮件正文和附件
4. 返回列表

### 发送邮件
1. 点击发送邮件 Tab
2. 填写收件人、主题、正文
3. 点击发送 → 显示发送结果

### 切换邮箱
1. 邮箱管理页 → 点击已有地址
2. 切换为活跃邮箱
3. 收件箱自动刷新该邮箱的邮件

## 颜色方案

| 用途 | 浅色模式 | 深色模式 |
|------|---------|---------|
| Primary (品牌色) | #2563EB (蓝色) | #3B82F6 |
| Background | #F8FAFC | #0F172A |
| Surface (卡片) | #FFFFFF | #1E293B |
| Foreground (文字) | #0F172A | #F1F5F9 |
| Muted (次要文字) | #64748B | #94A3B8 |
| Border | #E2E8F0 | #334155 |
| Success | #10B981 | #34D399 |
| Warning | #F59E0B | #FBBF24 |
| Error | #EF4444 | #F87171 |

设计采用 Slate 色系，搭配蓝色作为品牌主色调，营造专业、简洁的邮箱客户端感觉。

## 交互设计

- 邮件列表使用 FlatList，支持下拉刷新
- 邮件卡片点击有 opacity 反馈
- 发送邮件成功后有 haptic 反馈
- 删除操作需要确认
- 加载状态使用 ActivityIndicator
- 错误状态使用 toast 提示
